# 커머스 프로젝트

## 프로젝트 소개
여러 상품들을 장바구니에 담고 주문할 수 있는 간단한 커머스 프로그램

## 개발 목적
내일배움캠프 과제 & 공부

## 개발 인원
1인

## 개발 기간
1주일

## 요구 조건
- 객체 지향 설계를 적용 (캡슐화)
- 카테고리 별 상품 목록 출력
  - 출력 : 선택한 카테고리의 상품 정보
  - 담기 : 장바구니에 추가
  - 검색 필터 : 가격대 조건으로 상품을 필터링 할 수 있어야 함
- 장바구니 기능
  - 출력 : 장바구니 내 모든 상품 정보 (+ 재고, 총 금액)
  - 빼기 : 이름 검색해서 장바구니에서 해당 상품 제거
  - 비우기 : 장바구니 초기화
  - 주문하기 : 주문하기 (장바구니 초기화 및 재고 감소)
- 주문 기능
  - 할인 : 유저 등급 별 할인 정책 적용
- 관리자 기능
  - 관리자 비밀번호 인증 기능 (3회 실패 시, 메인 화면으로)
  - 상품 추가, 수정, 제거(제거 시 유저 장바구니에서 상품 제거 해야 함)


## 구현 내용
### 클래스 정의

#### System
- Main : 시작 지점이 되는 클래스
- CommerceSystem: 프로그램 비즈니스 로직 클래스 (유저 UI)

#### Model
- Product: 개별 상품 정보를 가지는 클래스
- Customer : 유저 정보를 가지는 클래스
- Order : 주문 정보를 가지는 클래스
- Discount : 할인 정보를 가지는 클래스 (Order에 의존성)
- ShoppingCard : 장바구니 정보를 가지는 클래스 (Customer에 의존성)

#### Manager (싱글톤 패턴)
- Category : Product를 관리하는 클래스 (ProductManager로 변경하고 싶지만 요구사항이라 변경하지 않음)
- CustomerManager : 유저를 관리하는 클래스
- OrderManager : 주문을 관리하는 클래스

#### Ploicy (전략 패턴)
- DiscountPolicy : 모든 정책에 적용되는 인터페이스
- BronzeDiscount, SilverDiscount, GoldDiscount, PlatinumDiscount : 각 등급 별 할인 정책
